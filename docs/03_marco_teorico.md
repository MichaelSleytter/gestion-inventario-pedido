# Marco Teórico

## 1. Sistemas de Gestión de Inventario

Un sistema de gestión de inventario es una herramienta tecnológica
que permite a las organizaciones controlar, registrar y monitorear
el flujo de productos dentro de su cadena de suministro. Según
Chopra y Meindl (2021), la gestión eficiente del inventario es
uno de los pilares fundamentales de la cadena de suministro, ya
que impacta directamente en la disponibilidad de productos y en
los costos operativos de la empresa.

Los sistemas modernos de inventario permiten automatizar procesos
que anteriormente se realizaban de forma manual, reduciendo
significativamente el margen de error y mejorando la toma de
decisiones en tiempo real. La implementación de estos sistemas
en distribuidoras comerciales ha demostrado reducir las pérdidas
por desabastecimiento hasta en un 35% según estudios recientes
del sector logístico latinoamericano.

## 2. Arquitectura REST y Spring Boot

La arquitectura REST (Representational State Transfer) es un estilo
arquitectónico para el diseño de servicios web que utiliza el
protocolo HTTP para la comunicación entre cliente y servidor.
Fielding (2000) definió REST como un conjunto de restricciones
arquitectónicas que, cuando se aplican correctamente, permiten
construir sistemas escalables, mantenibles e interoperables.

Spring Boot es un framework de Java que simplifica el desarrollo
de aplicaciones empresariales al proporcionar configuración
automática y un servidor embebido. Según Walls (2022), Spring Boot
permite a los desarrolladores enfocarse en la lógica de negocio
en lugar de la configuración de infraestructura, acelerando
significativamente el ciclo de desarrollo.

## 3. Control de Versiones con Git y GitHub

Git es un sistema de control de versiones distribuido creado por
Linus Torvalds en 2005. Permite a los equipos de desarrollo
trabajar de manera colaborativa, manteniendo un historial completo
de todos los cambios realizados en el código fuente. GitHub es
una plataforma basada en Git que agrega funcionalidades de
colaboración como Pull Requests, Issues y Actions.

Los Conventional Commits son una especificación para agregar
significado semántico a los mensajes de commit, facilitando
la generación automática de changelogs y la comprensión del
historial del proyecto (Conventional Commits, 2022).

## 4. Bases de Datos Relacionales y JPA/Hibernate

MySQL es uno de los sistemas de gestión de bases de datos
relacionales más utilizados en el mundo. Su modelo basado en
tablas, relaciones y claves foráneas permite estructurar la
información de manera coherente y eficiente. JPA (Java Persistence
API) es una especificación de Java que define cómo los objetos
Java se mapean a tablas de bases de datos relacionales.

Hibernate es la implementación más popular de JPA y proporciona
un ORM (Object-Relational Mapping) completo que permite a los
desarrolladores interactuar con la base de datos usando objetos
Java en lugar de sentencias SQL directas, mejorando la
mantenibilidad y portabilidad del código.

## 5. DevSecOps y CI/CD en proyectos universitarios

DevSecOps es una metodología que integra prácticas de seguridad
dentro del ciclo de vida del desarrollo de software desde sus
etapas más tempranas. La Integración Continua y el Despliegue
Continuo (CI/CD) son prácticas fundamentales de DevSecOps que
permiten automatizar la compilación, prueba y despliegue del
software mediante pipelines automatizados.

GitHub Actions es una plataforma de CI/CD integrada en GitHub
que permite definir flujos de trabajo automatizados mediante
archivos YAML. Su uso en proyectos universitarios permite a los
estudiantes familiarizarse con prácticas profesionales de la
industria del software.
